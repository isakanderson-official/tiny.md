# Use cases

tiny.md is best when you want a small site that is easy to edit, easy to move, and easy to understand later.

## Personal sites

Use it for a homepage, portfolio, resume, reading list, or link hub. The folder model keeps content simple without forcing a database or admin UI.

## Project pages

Use it for a small open source project, product notes, launch page, or changelog. Markdown keeps writing fast, and the generated output is just static HTML and assets.

## Lightweight docs

Use it for guides, onboarding notes, team handbooks, and small documentation sites. Navigation lives in `site/site.md`, so the shape of the site stays visible.

## Course and tutorial material

Use one folder per lesson. Keep screenshots, diagrams, and downloads next to the Markdown file that references them.

## When to choose something else

tiny.md is intentionally small. Choose a larger framework if you need user accounts, server-rendered data, complex interactive components, or a plugin ecosystem.
